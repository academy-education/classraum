# 결제

> **원장과 매니저**만 사용 가능. 교사는 결제 탭이 보이지 않습니다.

두 가지 결제 유형:

- **일회성 결제** — 교재, 추가 수수료 등 반복되지 않는 항목.
- **정기 결제** — 매월 고정 수강료 또는 지속적으로 청구되는 항목.

학원 수강료가 매월 변동되는 경우 **일회성 결제**만 사용하는 것을 권장합니다 — 청구서마다 정확한 금액을 담을 수 있기 때문입니다.

## 결제 대시보드

```mockup
payments-list
```

결제 탭에는 총 매출, 미수금, 활성 템플릿, MRR 통계 카드와 정렬/필터/행별 점 세 개 메뉴(편집/삭제)가 있는 청구서 표가 표시됩니다. 상단의 탭으로 **일회성 결제**와 **정기 결제**를 전환할 수 있습니다.

## 1단계 — 결제 플랜 만들기(정기 결제용)

"결제 플랜"은 재사용 가능한 템플릿입니다 — 예: *"월 수강료 – ₩400,000"*. 한 번 만든 후 학생을 배정합니다.

```mockup
add-payment-plan
```

1. **결제 플랜**으로 이동하세요.
2. 검색창 옆의 **결제 플랜 추가**를 클릭하세요.
   - ⚠️ **결제 추가** 버튼과 다릅니다. 혼동하지 마세요.
3. **플랜 이름**을 입력하세요(예: *월 수강료*).
4. **금액**을 입력하고 **반복 유형**(주간 또는 월간)을 선택하세요.
5. **결제일**을 설정하세요(예: 매월 25일).
   - 매 주기마다 해당 일자에 시스템이 자동으로 청구를 생성하고 카카오톡으로 학생/학부모에게 알립니다(앱 내 알림은 향후 업데이트 예정).
6. 기록용 **시작일**을 선택하세요. **종료일**은 선택사항입니다.
7. **결제 플랜 추가**를 클릭하세요.

## 2단계 — 학생을 플랜에 배정

```mockup
assign-recurring-payment
```

플랜이 하나 이상 있으면:

1. **정기 결제**로 이동하세요.
2. 우측 상단의 **결제 추가**를 클릭하세요.
3. 결제 유형으로 **정기**를 선택하세요.
4. 만들어 둔 **결제 플랜**을 선택하세요 — 파란색 도움말 카드에 금액과 다음 청구일이 표시됩니다.
5. 적용할 학생을 선택하세요 — 모든 학생에게 같은 수강료라면 **전체 선택**을 사용하세요.
6. **결제 추가**를 클릭하세요.

선택한 모든 학생이 정기 결제 일정에 등록됩니다. 이후 정기 결제 페이지에서 개별 항목을 편집, 조회, 제거할 수 있습니다.

## 3단계 — 일회성 결제 만들기

```mockup
add-one-time-payment
```

일회성 청구(교재, 비품, 연체료 등)의 경우:

1. 우측 상단의 **결제 추가**를 클릭하세요.
2. 결제 유형으로 **일회성**을 선택하세요.
3. **청구서 이름**을 입력하세요.
4. 청구할 **학생**을 선택하세요.
5. **금액**과 필요시 **할인**(사유 포함)을 입력하세요.
6. **결제 마감일**을 설정하세요 — *필수*. 마감일이 지나면 결제 링크가 자동으로 만료되어 학생/학부모가 결제할 수 없습니다.
7. 필요시 **결제 수단**과 **결제 상태**를 설정하세요(현금/대면 결제 기록용).
8. **결제 추가**를 클릭하세요.

생성 후 목록에서 청구서를 열어 세부사항을 보거나 수정할 수 있습니다.

## 학부모와 학생이 보는 화면

결제를 만들면 수신자에게 결제 링크가 포함된 알림이 전송됩니다. 다음 방법으로 결제할 수 있습니다:

- 신용카드
- 카카오페이
- 네이버페이
- 토스

거래가 자동으로 기록되고 청구서 상태가 결제 완료로 변경됩니다.
